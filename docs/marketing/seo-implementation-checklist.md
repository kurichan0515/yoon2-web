## 📖 用語・施策の説明

### パンくずリスト（Breadcrumb）とは？

**パンくずリスト**は、今いるページがサイトのどこにあるかを示すナビゲーションです。

- **表示例**: `ホーム > メニュー > 耳つぼメニュー` のように、階層を「 Home → 大分類 → 今のページ 」で表示します。
- **役割**:
  - **ユーザー**: どこにいるか分かる・ひとつ上の階層に戻りやすい。
  - **SEO**: 検索結果にパンくずが表示されることがあり、クリック率の向上やサイト構造の理解に役立ちます。
- **このサイトの場合**: トップ（/）・プライバシー（/privacy）・SNS（/sns）など、階層が浅いため必須ではありません。メニューや予約など「階層が深いページ」を増やしたときに導入を検討するとよいです。

---

### モバイル最適化とは？

**モバイル最適化**は、スマートフォンやタブレットでサイトが正しく・快適に使えるようにする施策です。

- **主な内容**:
  1. **レスポンシブデザイン**  
     画面幅に合わせてレイアウトが変わり、スマホでも読みやすく操作しやすい表示にすること。
  2. **モバイルフレンドリーテスト**  
     Google の基準で「スマホ向けに問題ないか」を判定。合格するとモバイル検索で不利になりにくい。
  3. **表示速度**  
     スマホでは通信が遅いことが多いため、画像の軽量化・不要な読み込みの削減などで表示を速くする。
  4. **タッチ操作**  
     ボタンやリンクを指で押しやすいサイズ・間隔にすること。
- **確認方法**:  
  Chrome の開発者ツールで「スマホ表示」に切り替えて見た目を確認する、または [Google モバイルフレンドリーテスト](https://search.google.com/test/mobile-friendly) や Lighthouse で「モバイル」を計測する。
- **このサイト**: レスポンシブ・タッチターゲット 44px・Safe Area・touch-action・フォーム 16px 等を実装済み。公式のモバイルフレンドリーテストや Lighthouse での計測は随時推奨。
