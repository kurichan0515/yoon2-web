# 🔐 管理者アクセス情報

## 📋 概要

yoon² 管理システムへのアクセス情報と使用方法をまとめたドキュメントです。

## 🚀 管理画面へのアクセス

### **管理画面URL**
```
https://your-domain.com/?admin=login
```

### **直接アクセス**
- ホームページ: `https://your-domain.com/`
- 管理画面: `https://your-domain.com/?admin=login`

## 🔑 ログイン情報

### **初期管理者アカウント**
- **メールアドレス**: `admin@yoon2.com`
- **パスワード**: `admin123456`

### **⚠️ セキュリティ注意事項**
- 本番環境では必ずパスワードを変更してください
- 強力なパスワード（12文字以上、英数字記号混在）を使用してください
- 管理者アカウントの管理を適切に行ってください

## 🛠️ 管理者アカウントの初期設定

### **開発環境での設定**
```javascript
// ブラウザのコンソールで実行
window.setupAdmin()
```

### **手動での管理者アカウント作成**
1. Firebase Console にアクセス
2. Authentication > Users でユーザーを作成
3. Firestore で `admins` コレクションに管理者情報を追加

```javascript
// Firestore に追加する管理者情報
{
  email: "admin@yoon2.com",
  displayName: "yoon² 管理者",
  isAdmin: true,
  createdAt: "2024-01-01T00:00:00Z",
  lastLogin: null
}
```

## 📊 管理画面機能

### **1. メインダッシュボード**
- **URL**: `/admin-dashboard`
- **機能**: 管理機能メニュー、システム情報表示

### **2. アナリティクス**
- **URL**: `/analytics`
- **機能**: ページビュー分析、統計情報表示

### **3. 予約詳細**
- **URL**: `/admin-booking-details`
- **機能**: Googleカレンダー連携、予約情報表示

### **4. 店舗情報**
- **URL**: `/shop`
- **機能**: 店舗基本情報の確認

## 🔒 セキュリティ機能

### **認証システム**
- Firebase Authentication を使用
- メール・パスワード認証
- 管理者権限のFirestore管理

### **アクセス制御**
- 管理者以外は管理画面にアクセス不可
- 認証状態のリアルタイム監視
- 適切なセッション管理

### **権限管理**
- Firestore の `admins` コレクションで管理者権限を管理
- ログイン時に管理者権限を確認
- 権限のないユーザーは自動的にホームページにリダイレクト

## 📱 管理画面の使用方法

### **1. ログイン手順**
1. `/?admin=login` にアクセス
2. メールアドレスとパスワードを入力
3. 「ログイン」ボタンをクリック
4. 管理者ダッシュボードに自動遷移

### **2. ログアウト手順**
1. 管理画面右上の「ログアウト」ボタンをクリック
2. 確認ダイアログで「OK」を選択
3. ホームページに自動遷移

### **3. 機能別アクセス**
- **アナリティクス**: ダッシュボードの「アナリティクスを見る」ボタン
- **予約詳細**: ダッシュボードの「予約詳細を見る」ボタン
- **店舗情報**: ダッシュボードの「店舗情報を確認」ボタン

## 🚨 トラブルシューティング

### **ログインできない場合**
1. メールアドレスとパスワードを確認
2. 管理者権限がFirestoreに正しく設定されているか確認
3. Firebase Authentication の設定を確認

### **管理画面にアクセスできない場合**
1. 認証状態を確認
2. 管理者権限の設定を確認
3. ブラウザのキャッシュをクリア

### **エラーメッセージ**
- `auth/user-not-found`: ユーザーが見つかりません
- `auth/wrong-password`: パスワードが間違っています
- `auth/invalid-email`: メールアドレスの形式が正しくありません
- `auth/too-many-requests`: ログイン試行回数が多すぎます

## 📞 サポート

### **技術的な問題**
- Firebase Console で認証状態を確認
- Firestore で管理者権限を確認
- ブラウザの開発者ツールでエラーを確認

### **セキュリティに関する問題**
- 管理者アカウントのパスワードを変更
- 不要な管理者アカウントを削除
- アクセスログを定期的に確認
