# カスタムドメイン設定ガイド

`yoon2.com` をFirebase Hostingに設定する手順です。

## 前提条件

- ✅ ドメイン `yoon2.com` を取得済み
- ✅ Firebaseプロジェクト `react-firebase-app-2025` が設定済み
- ✅ Firebase Hostingが有効化済み

## 設定手順

### ステップ1: Firebase Consoleでカスタムドメインを追加

1. **Firebase Consoleにアクセス**
   - https://console.firebase.google.com/
   - プロジェクト `react-firebase-app-2025` を選択

2. **Hostingセクションを開く**
   - 左メニューから「Hosting」を選択

3. **カスタムドメインを追加**
   - 「カスタムドメインを追加」ボタンをクリック
   - ドメイン名に `yoon2.com` を入力
   - 「続行」をクリック

### ステップ2: DNS設定

Firebase Consoleに表示されるDNSレコードを、ドメイン管理画面で設定します。

#### 表示されるDNSレコード例

通常、以下のパターンが表示されます：

**パターンA: Aレコード + TXTレコード（推奨）**
```
タイプ: A
名前: @ (または空白)
値: 199.36.158.100

タイプ: TXT
名前: @ (または空白)
値: hosting-site=react-firebase-app-2025
```

**パターンB: CNAMEレコード + TXTレコード**
```
タイプ: CNAME
名前: @ (または空白)
値: react-firebase-app-2025.web.app

タイプ: TXT
名前: @ (または空白)
値: hosting-site=react-firebase-app-2025
```

**重要**: Firebase Hostingでは、ドメイン検証のためにTXTレコードが必須です。AレコードまたはCNAMEレコードと一緒に設定してください。

#### DNS設定手順（お名前.comの場合）

1. **お名前.comにログイン**
   - https://www.onamae.com/ にアクセス
   - 右上の「会員ログイン」をクリック
   - 会員IDとパスワードでログイン

2. **ドメイン管理画面を開く**
   - トップページから「ドメイン」→「ドメイン設定」をクリック
   - または、直接「ドメイン設定」メニューを選択
   - 設定したいドメイン（例: `yoon2.com`）を選択

3. **DNSレコード設定画面を開く**
   - ドメイン詳細画面で「DNS関連機能の設定」をクリック
   - または「ネームサーバー設定」から「DNSレコード設定を利用する」を選択
   - 「DNSレコード設定を利用する」にチェックを入れる（初回の場合）

4. **DNSレコードを追加**
   
   **パターンA: Aレコードの場合**
   - 「レコード追加」ボタンをクリック
   - 以下の情報を入力：
     - **ホスト名**: `@` または空白（ルートドメインの場合）
     - **TYPE**: `A` を選択
     - **VALUE**: Firebase Consoleに表示されたIPアドレス（例: `199.36.158.100`）
     - **TTL**: `3600`（デフォルト値でOK）
   - 「追加」ボタンをクリック
   - 複数のAレコードがある場合は、それぞれ追加（例: `151.101.65.195` も追加）

   **パターンB: CNAMEレコードの場合**
   - 「レコード追加」ボタンをクリック
   - 以下の情報を入力：
     - **ホスト名**: `@` または空白（ルートドメインの場合）
     - **TYPE**: `CNAME` を選択
     - **VALUE**: Firebase Consoleに表示された値（例: `react-firebase-app-2025.web.app`）
       - ⚠️ 注意: 末尾のピリオド（`.`）は不要です
     - **TTL**: `3600`（デフォルト値でOK）
   - 「追加」ボタンをクリック

   **重要: TXTレコードも追加が必要です**
   - Firebase ConsoleでTXTレコードが表示されている場合、必ず追加してください
   - 「レコード追加」ボタンをクリック
   - 以下の情報を入力：
     - **ホスト名**: `@` または空白（ルートドメインの場合）
     - **TYPE**: `TXT` を選択
     - **VALUE**: Firebase Consoleに表示された値（例: `hosting-site=react-firebase-app-2025`）
       - ⚠️ 注意: 値はそのままコピー&ペーストしてください
     - **TTL**: `3600`（デフォルト値でOK）
   - 「追加」ボタンをクリック

5. **設定を保存**
   - 設定内容を確認
   - 「設定する」または「保存」ボタンをクリック
   - 確認画面で「設定する」をクリック

6. **設定確認**
   - DNSレコード一覧に追加したレコードが表示されることを確認
   - 既存のレコードと競合していないか確認（必要に応じて削除）

**重要事項:**
- お名前.comでは、ルートドメイン（`@`）にCNAMEレコードを設定できない場合があります
- その場合は、Firebase ConsoleでAレコード方式を選択してください
- 設定変更後、反映まで数分〜数時間かかることがあります

#### DNS設定手順（ムームードメインの場合）

1. **ムームードメインにログイン**
   - https://muumuu-domain.com/ にアクセス

2. **DNS設定画面を開く**
   - 「ドメイン操作」→「DNS関連機能の設定」
   - `yoon2.com` を選択

3. **DNSレコードを追加**
   - Firebase Consoleに表示されたレコードを設定
   - 保存

### ステップ3: DNS伝播の確認

DNS設定後、反映まで数分〜48時間かかることがあります。

#### 確認方法

```bash
# DNSレコードの確認
nslookup yoon2.com

# または
dig yoon2.com
```

Firebase Consoleでも自動的に検証が行われます。

### ステップ4: SSL証明書の自動発行

DNS設定が正しく反映されると、Firebaseが自動でLet's EncryptのSSL証明書を発行します。

- **所要時間**: 数分〜数時間
- **ステータス**: Firebase ConsoleのHostingセクションで確認可能
- **完了**: 「証明書がアクティブです」と表示されれば完了

### ステップ5: 動作確認

1. **ブラウザでアクセス**
   - `https://yoon2.com` にアクセス
   - SSL証明書が正しく表示されることを確認

2. **リダイレクトの確認**
   - `http://yoon2.com` → `https://yoon2.com` に自動リダイレクトされることを確認

## トラブルシューティング

### DNS設定が反映されない

- **確認**: DNS設定が正しく保存されているか
- **待機**: 最大48時間かかる場合があります
- **確認コマンド**: `nslookup yoon2.com` で確認

### お名前.com特有の問題

#### ルートドメインにCNAMEレコードが設定できない

**問題**: お名前.comでは、ルートドメイン（`@`）にCNAMEレコードを設定できない場合があります。

**解決方法**:
1. Firebase Consoleで「Aレコード」方式を選択してください
2. Firebase Consoleでカスタムドメインを追加する際に、Aレコード方式を選択します
3. 表示されたIPアドレスを、お名前.comでAレコードとして設定します

#### DNSレコード設定画面が表示されない

**問題**: 「DNSレコード設定を利用する」オプションが表示されない

**解決方法**:
1. お名前.comの「ネームサーバー設定」を確認
2. デフォルトのネームサーバーを使用している場合、DNSレコード設定が利用可能です
3. カスタムネームサーバーを使用している場合、そのネームサーバー側で設定が必要です

#### 設定が保存されない

**問題**: DNSレコードを追加しても保存されない

**解決方法**:
1. ブラウザのキャッシュをクリアして再試行
2. 別のブラウザで試す
3. お名前.comのサポートに問い合わせ

#### 既存のレコードと競合する

**問題**: 既存のAレコードやCNAMEレコードがある

**解決方法**:
1. 既存のレコードを削除してから、新しいレコードを追加
2. または、既存のレコードを編集して値を変更
3. 重要なレコード（メールサーバーなど）がある場合は、事前にバックアップを取る

### SSL証明書が発行されない

- **確認**: DNS設定が正しく反映されているか
- **待機**: 数時間かかる場合があります
- **再試行**: Firebase Consoleで「再試行」ボタンをクリック

### アクセスできない

- **確認**: Firebase Hostingにデプロイされているか
- **確認コマンド**: `firebase hosting:sites:list`
- **デプロイ**: `npm run build && firebase deploy --only hosting`

## デプロイ手順

カスタムドメイン設定後、通常通りデプロイします：

```bash
# 1. ビルド
npm run build

# 2. デプロイ
firebase deploy --only hosting

# 3. 確認
firebase hosting:sites:list
```

## 注意事項

- **wwwサブドメイン**: `www.yoon2.com` も設定したい場合は、別途追加が必要です
- **DNS伝播時間**: 設定変更後、反映まで時間がかかることがあります
- **SSL証明書**: 自動更新されるため、手動での更新は不要です
- **お名前.comの制限**: ルートドメイン（`@`）にCNAMEレコードを設定できない場合があるため、Aレコード方式を推奨します
- **メール設定**: 既存のメール設定がある場合、DNSレコード変更時に影響が出る可能性があります

## お名前.comでの設定の流れ（まとめ）

1. ✅ Firebase Consoleでカスタムドメインを追加
2. ✅ Firebase Consoleに表示されたDNSレコード情報をメモ
3. ✅ お名前.comにログイン
4. ✅ ドメイン設定 → DNSレコード設定画面を開く
5. ✅ Firebase Consoleの指示に従ってDNSレコードを追加
6. ✅ 設定を保存
7. ✅ DNS伝播を待つ（数分〜48時間）
8. ✅ Firebase ConsoleでSSL証明書の発行を確認
9. ✅ ブラウザでアクセスして動作確認
