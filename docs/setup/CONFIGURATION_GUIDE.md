# 設定変更ガイド

このガイドでは、店舗情報、料金、SNSリンクなどの設定を簡単に変更する方法を説明します。

## 🚀 クイックスタート

### 1. 環境変数ファイルを作成
```bash
cp env.example .env
```

### 2. 設定を変更
`.env`ファイルを編集して、必要な情報を更新します。

### 3. アプリを再起動
設定を反映するためにアプリケーションを再起動します。

## 🏠 店舗引っ越し時の変更手順

### 1. 準備
- [ ] 新しい住所を確認
- [ ] 新しい最寄駅・アクセス情報を確認
- [ ] 駐車場情報を確認
- [ ] 営業時間に変更がないか確認

### 2. 環境変数を更新
`.env`ファイルで以下を更新：

```bash
# 新しい住所
REACT_APP_SHOP_ADDRESS=新しい住所

# 新しいアクセス情報
REACT_APP_SHOP_ACCESS_STATION1=新しい最寄駅1
REACT_APP_SHOP_ACCESS_STATION2=新しい最寄駅2
REACT_APP_SHOP_ACCESS_STATION3=新しいバス停
REACT_APP_SHOP_ACCESS_LANDMARKS=新しい目印
REACT_APP_SHOP_ACCESS_PARKING=新しい駐車場情報
```

### 3. 動作確認
- [ ] アプリを再起動
- [ ] 店舗情報ページで住所が正しく表示されているか確認
- [ ] フッターの住所が更新されているか確認

## 💸 料金変更時の手順

### 1. 変更する料金を特定
どのメニューの料金を変更するか確認

### 2. 環境変数を更新
該当する環境変数を変更：

```bash
# 例：耳つぼ新規価格を4500円に変更
REACT_APP_PRICE_MIMITUBO_NEW=4500
```

### 3. 動作確認
- [ ] アプリを再起動
- [ ] ホームページのメニュー表示で料金が更新されているか確認
- [ ] 予約ページで料金が正しく表示されているか確認

## 🔧 高度な設定変更

### 新しいメニューを追加する場合

1. `src/config/appConfig.js`の`services`配列に新しいメニューを追加
2. 環境変数が必要な場合は`env.example`にも追加
3. 該当コンポーネントでメニューが表示されるか確認

### 完全に新しい情報項目を追加する場合

1. `env.example`に新しい環境変数を追加
2. `src/config/appConfig.js`で環境変数を読み込み
3. 必要なコンポーネントで新しい設定を使用

---

## ⚠️ 注意事項

1. **環境変数の名前**: `REACT_APP_`で始まる必要があります
2. **再起動が必要**: 環境変数を変更したら必ずアプリを再起動してください
3. **バックアップ**: 変更前に現在の`.env`ファイルをバックアップしてください
4. **料金は数字のみ**: 料金設定では数字のみを入力してください（円マークは不要）

## 🆘 トラブルシューティング

### Q: 設定を変更したのに反映されない
A: アプリケーションを再起動してください。環境変数の変更は再起動が必要です。

### Q: 料金が0円で表示される
A: 環境変数に数字以外の文字が含まれていないか確認してください。

### Q: 画面が崩れた
A: 長すぎるテキストが原因の可能性があります。適切な長さに調整してください。

## 🔧 新機能: 設定検証

アプリケーション起動時に自動的に設定を検証し、問題があれば開発者コンソールに表示します。

### 検証内容
- ✅ 必須項目の設定チェック
- ✅ 電話番号形式の検証
- ✅ 営業時間の妥当性チェック
- ✅ 料金設定の検証
- ✅ SNSリンクのURL形式チェック

### 使用方法
1. 開発モードでアプリを起動
2. ブラウザの開発者ツールを開く
3. コンソールタブで "🔧 Configuration Validation" の項目を確認
